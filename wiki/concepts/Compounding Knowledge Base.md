# Compounding Knowledge Base

**One-line definition:** A knowledge base that grows richer with each addition because connections are maintained incrementally rather than re-derived on demand.

## Explanation

A compounding knowledge base is one where:
- Each new source strengthens existing pages with more evidence
- Cross-references and connections are built up over time, not recreated per query
- Contradictions between sources are tracked and visible
- Synthesis pages represent genuine intellectual work that doesn't disappear into chat history

The analogy from [[sources/llm-wiki-pattern]]: fan wikis like Tolkien Gateway — thousands of interlinked pages built by a community over years. The LLM Wiki pattern achieves this individually, with the LLM doing the maintenance work that communities usually spread across many contributors.

The reason human-maintained wikis fail: maintenance burden grows faster than value. Cross-referencing, updating stale claims, noting contradictions — these are tedious and humans abandon them. LLMs don't get bored and can touch 15 files in one pass.
